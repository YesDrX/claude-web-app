# Claude Code Web

A web-based UI for [Claude Code](https://claude.ai/code), built with FastAPI. Manage multiple Claude sessions, chat in real-time via WebSocket, upload files, browse generated reports, and configure permission profiles — all from your browser.

## Features

- **Multi-session dashboard** — create, rename, search, and delete Claude sessions
- **Real-time chat** — WebSocket streaming with markdown rendering, tool call visibility, and live status
- **Session resume** — reconnect to active or completed sessions with full history replay
- **File upload** — attach files to prompts via button or drag-and-drop; files land in `reports/chat_files/`
- **Reports browser** — view and download files generated by Claude in `reports/`
- **Permission profiles** — switch between "Fully Autonomous" and restricted modes per session
- **Templates** — save and reuse session configurations
- **Import** — scan and import existing Claude CLI sessions from your history
- **Mobile-friendly** — responsive design works on phones and tablets

## Quick Start

### Prerequisites

- Python 3.11+
- [Claude Code](https://claude.ai/code) CLI installed and authenticated

### Install

```bash
git clone https://github.com/YesDrX/claude-code-web.git
cd claude-code-web
python -m venv .venv
.venv\Scripts\activate   # Windows
# source .venv/bin/activate  # macOS/Linux
pip install -r requirements.txt
```

### Configure

```bash
copy config.example.toml config.toml   # Windows
# cp config.example.toml config.toml   # macOS/Linux
```

Edit `config.toml`:

- Set `auth.username` and `auth.password_hash` (generate with `python change_password.py your-password`)
- Set `claude.executable_path` to your Claude CLI location
- Adjust other paths to match your system

### Run

```bash
python main.py
# Or: uvicorn main:app --host 0.0.0.0 --port 8000
```

Open `http://localhost:8000` and log in.

```

├── auth.py              # Authentication (pbkdf2 password hashing)
├── database.py          # SQLite/aiosqlite data layer
├── models.py            # Pydantic request/response models
├── change_password.py   # Password hash generator
├── config.example.toml  # Example configuration
├── requirements.txt     # Python dependencies
├── templates/           # Jinja2 HTML templates
│   ├── base.html
│   ├── chat.html        # Real-time chat interface
│   ├── index.html       # Session dashboard
│   ├── login.html
│   ├── reports.html
│   ├── settings.html
│   └── templates_manage.html
├── static/              # Static assets
└── reports/             # Generated files (gitignored)
    └── chat_files/      # Uploaded chat files (gitignored)
```
